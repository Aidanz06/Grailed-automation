Workspace at the docked-Chrome width (900px)

The sidebar is a fixed 320px column (App.tsx:496). "Dock Chrome" shrinks the app window toward its 900px minimum (ui/main.js:612, 642) — exactly when the seller is filling and needs the editor most, the editor pane drops to ~580px and the right rail is already stacked below the form (rail breakpoint is lg/1024px). Proposed: the sidebar collapses to a ~116px photo rail below ~1100px — large 4:5 photo tiles (same aspect as the board cards) so each draft stays instantly recognizable and one click away; the photo itself is the navigation. Titles hide, but the readiness state survives as a compact chip on every tile. The editor gains ~200px, and the full sidebar returns at normal widths. A toggle button (◧) sits in the workspace header next to "← Home" — the seam between sidebar and editor — so you can compact/expand manually at any width; a manual choice overrides the automatic breakpoint and persists (tailor.sidebarMode: auto · expanded · compact).

Error boundary fallback

There is no React error boundary anywhere in ui/src. Any uncaught render error unmounts the whole tree — and the packaged app has no visible console (the codebase itself documents this at lib/utils.ts:40-44), so the seller sees a dead white window with no recourse. Proposed: an App-root boundary (plus one around the Editor pane) that keeps the chrome alive, shows the real error message per the app's own "every failure must carry the real message" rule, and offers Copy + Reload. Copy follows the existing left-accent callout convention.

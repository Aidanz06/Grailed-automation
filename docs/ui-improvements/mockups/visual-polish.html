Visual polish: price formatting + thumbnail fallback color

Two small, purely visual fixes. 1) money() is defined four times as '$' + n — no thousands separator, so "$1200" appears on board cards, list rows, and the big serif price. 2) The photo-tint fallback is a hardcoded '#333' in 7 files — in the light theme it renders as a heavy near-black chip that ignores the warm-paper palette. Proposed: one shared money() using toLocaleString, and the fallback becomes bg-muted (theme-aware).

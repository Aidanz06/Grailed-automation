# 03 — Removal & Change Safety Manifest

Scope: every element a Phase-2 proposal removes or alters. Search evidence is from ripgrep over the whole repo excluding `node_modules/`, `dist/` (build output, regenerated by `npm run ui:build`), and `.git/`. General findings that bound the blast radius:

- **No UI tests, no `data-testid`, no E2E scripts target the renderer.** `phase0Test/*.js` drives *Grailed's* pages via CDP, never the app UI (verified: no hits for `localhost:5173`/`#root`/app selectors).
- **No analytics, no CMS, no `dangerouslySetInnerHTML`** (verified). Only two template-literal classNames exist (`PhotoRow.tsx:28`, `BatchProgressBar.tsx:69`) and neither builds class names from data — the M-3 codemod cannot miss a dynamically-built `text-[Npx]`.
- **No lazy `import()` anywhere in `ui/src`** — static import graph is the whole graph.
- **Tailwind content globs** cover only `ui/index.html` + `ui/src/**` (`tailwind.config.cjs:7`) — no external stylesheet consumers.
- **`ui/main.js` (Electron main) never touches renderer DOM/classes** (no `querySelector`/`getElementById` hits).
- The static design mockups in `design/*.html` reference old markup but are documentation, not runtime — never a blocker.
